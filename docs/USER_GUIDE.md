## 一、系统功能介绍

### 1.1 系统是什么

TdxQuant 是一套 **A 股量化选股与监控预警系统**，核心帮助你：

- 📊 **选股**：用 5 大量化策略自动筛选股票，秒级输出候选清单
- 📡 **监控**：实时盯盘，条件触发自动推送预警到多通道
- 📈 **回测**：历史数据验证策略表现，输出夏普 / 胜率等指标
- 🤖 **半自动交易**：预警直达通达信客户端，人工确认下单

### 1.2 五大策略

| 策略 | 图标 | 说明 | 适用场景 |
|------|------|------|----------|
| **打板求涨停** | 🔥 | 筛选当日涨停或接近涨停的强势股 | 短线追涨打板 |
| **趋势主升浪** | 📈 | 均线多头排列 + MACD 金叉的趋势股 | 中线趋势跟踪 |
| **错杀低吸** | 🩹 | 短期超跌但基本面良好的股票 | 抄底反弹 |
| **弱转强** | ⚡ | 前一日弱势今日转强的股票 | 反转埋伏 |
| **强转弱反抽** | 🔄 | 强势股短期回调后的反抽机会 | 回调买入 |

### 1.3 系统界面（5 大模块）

系统采用单页应用，顶部 5 个 Tab 切换：

| Tab | 功能 |
|-----|------|
| **实时大屏** 📊 | 行情总览 + 涨跌榜 + 资金流向 + 策略胜率排行 |
| **策略管理** 🎯 | 策略启停 / 运行 / 复制 / 删除 / 编辑 |
| **选股结果** 📋 | 历史选股记录 + 导出 CSV/Excel + 回测入口 |
| **信号中心** 🔔 | 预警信号列表 + 详情抽屉 + 重推 |
| **板块管理** 🗂️ | 策略↔板块映射 + 成份股维护 + 批量导出 |

### 1.4 顶部工具栏

| 按钮 | 功能 |
|------|------|
| 🔍 **全局搜索** | Cmd+K / Ctrl+K 快速搜索策略 / 股票 / 信号 |
| 🔔 **通知中心** | 操作反馈与系统通知（未读红点提示） |
| ▶️ **运行全部** | 一键运行所有已启用策略 |
| 🔄 **热加载配置** | 修改 YAML 配置后无需重启即可生效 |
| 🌓 **主题切换** | 明 / 暗主题 |
| ⚙️ **推送通道配置** | 配置通达信 / 飞书 / Web / CSV 通道 |

## 二、快速上手（3 分钟）

### Step 1：打开系统

- 通过 **预览面板**（沙箱环境右侧）访问系统
- 可点 **「Open in New Tab」** 在新标签页打开

> ⚠️ 请勿直接访问 `localhost:3000` 等内部地址，这些是系统内部端口。

### Step 2：熟悉界面

1. 看顶部 **状态栏**：引擎状态 / 适配器模式（Mock 或 Real）/ 监控数 / 今日信号
2. 切换 5 个 Tab，了解每个模块
3. 点 🔔 通知中心，查看系统反馈

### Step 3：运行第一个策略

1. 切到 **策略管理** Tab
2. 找到「打板求涨停 🔥」策略
3. 确认开关为 **启用** 状态（绿色）
4. 点 **▶️ 运行** 按钮
5. 切到 **选股结果** Tab 查看结果
6. 点 **导出** 下载 CSV 或 Excel

### Step 4：查看实时行情

1. 切到 **实时大屏** Tab
2. 查看 **涨跌幅 Top3** 榜单
3. 查看 **资金流向** 三列（主力净流入 / 大买占比 / 换手率）
4. 查看 **策略胜率排行**

### Step 5：配置预警推送

1. 点顶部 ⚙️ **推送通道配置**
2. 按需开启通道（CSV 日志默认开启）
3. 保存配置

完成！你已掌握系统基本用法。详细操作见下一章。

## 三、用户权限说明

### 3.1 用户角色

TdxQuant 当前为 **单用户系统**（无登录认证），所有用户共享同一套配置。规划中的角色：

| 角色 | 权限 | 当前状态 |
|------|------|----------|
| **管理员** | 全部操作（策略增删改 / 配置 / 通道 / 回测） | ✅ 已实现（当前所有用户） |
| **研究员** | 策略查看 / 运行 / 回测，不可改配置 | 🔜 规划中 |
| **观察者** | 仅查看大屏与信号，不可操作 | 🔜 规划中 |

### 3.2 操作权限矩阵

| 操作 | 管理员 | 研究员（规划） | 观察者（规划） |
|------|--------|----------------|----------------|
| 查看实时大屏 | ✅ | ✅ | ✅ |
| 查看信号中心 | ✅ | ✅ | ✅ |
| 运行策略 | ✅ | ✅ | ❌ |
| 启用/禁用策略 | ✅ | ❌ | ❌ |
| 新增/复制/删除策略 | ✅ | ❌ | ❌ |
| 修改策略阈值 | ✅ | ❌ | ❌ |
| 导出 CSV/Excel | ✅ | ✅ | ❌ |
| 回测 | ✅ | ✅ | ❌ |
| 推送通道配置 | ✅ | ❌ | ❌ |
| 热加载配置 | ✅ | ❌ | ❌ |
| 板块管理 | ✅ | ❌ | ❌ |

### 3.3 数据权限

- 所有用户可见全部策略与历史数据
- 导出文件（CSV/Excel）保存在服务端 `data/csv/` 和 `data/excel/`
- 信号历史保存在 DuckDB，可通过 API 查询

## 四、操作流程详解

### 4.1 实时大屏 Tab 📊

实时大屏是系统首页，展示市场总览。

#### 4.1.1 顶部统计卡片

| 卡片 | 含义 |
|------|------|
| 监控股票数 | 当前订阅的股票数量 |
| 今日信号 | 今日触发的预警信号数 |
| 启用策略 | 已启用策略数 / 总策略数 |
| 引擎运行时间 | FastAPI 启动至今的时长 |

#### 4.1.2 涨跌幅 Top3 榜单

- **涨幅榜**（红色调）：当日涨幅最大的 3 只
- **跌幅榜**（绿色调）：当日跌幅最大的 3 只
- 每只显示：代码 / 名称 / 现价 / 涨跌幅

#### 4.1.3 资金流向三列（R7 新增）

| 列 | 指标 | 含义 |
|----|------|------|
| 左 | **主力净流入**（红色） | 大单买入净额，正值为主力净买入 |
| 中 | **大买占比**（琥珀） | 大买单占总成交的比例 |
| 右 | **换手率**（青色） | 当日成交量/流通股本 |

每列展示 Top 5，排名徽章 🥇🥈🥉。

#### 4.1.4 策略胜率排行

展示各策略的历史回测胜率排行（需先运行回测才有数据）。

#### 4.1.5 操作

- **手动刷新**：点刷新按钮立即更新
- **自动刷新**：每 10 秒自动轮询行情，15 秒轮询状态

---

### 4.2 策略管理 Tab 🎯

管理 5 大策略的启停、运行、复制、删除。

#### 4.2.1 策略卡片

每张卡片包含：

| 元素 | 说明 |
|------|------|
| 图标 + 名称 | 如 🔥 打板求涨停 |
| 策略 ID | 如 `dbqzt` |
| 板块代码 | 如 `ZD_DBQZT01` |
| 启用开关 | 绿色=启用，灰色=禁用 |
| ▶️ 运行 | 立即执行选股 |
| 📋 复制 | 复制策略创建新策略 |
| 🗑️ 删除 | 删除策略（启用中不可删） |
| 状态徽章 | 启用/禁用/运行中/已完成 |

#### 4.2.2 运行策略

1. 点策略卡片的 **▶️ 运行** 按钮
2. 等待执行（通常 2–10 秒）
3. 右上角通知中心显示结果
4. 切到 **选股结果** Tab 查看明细

#### 4.2.3 复制策略（创建新策略）

1. 点策略卡片的 **📋 复制** 按钮
2. 在弹窗中填写：
   - **新策略 ID**：拼音首字母小写（如 `mystr`），2–30 字符
   - **新策略名称**：中文名（如「我的策略」）
   - **Emoji**：UI 显示图标
3. 下方 **YAML 预览** 自动更新（可看到 `strategy_id` / `name` / `emoji` / `sector.code` 已替换）
4. 点 **确认创建**
5. 新策略默认 **禁用**，需手动启用

#### 4.2.4 删除策略

1. 点策略卡片的 **🗑️ 删除** 按钮
2. 弹窗显示警告（此操作不可恢复）
3. 在输入框 **完整输入策略 ID** 确认
4. 点 **确认删除**

> ⚠️ **启用中的策略不可删除**，需先关闭启用开关。

#### 4.2.5 批量操作

- **运行全部**：顶部按钮，一键运行所有已启用策略
- **启用全部 / 禁用全部**：批量切换启用状态

---

### 4.3 选股结果 Tab 📋

查看历史选股记录，导出数据，进入回测。

#### 4.3.1 结果列表

每次运行策略生成一条记录：

| 字段 | 说明 |
|------|------|
| 运行 ID | 唯一标识 |
| 策略 | 使用的策略名 |
| 运行时间 | 执行时间 |
| 结果数 | 选出的股票数量 |
| 耗时 | 执行秒数 |
| 操作 | 查看详情 / 导出 / 回测 |

#### 4.3.2 查看详情

点 **详情** 展开选股结果表格：

| 字段 | 说明 |
|------|------|
| 代码 | 股票代码（如 600519） |
| 名称 | 股票名称 |
| 评分 | 策略评分（越高越优） |
| 现价 | 当前价格 |
| 涨跌幅 | 当日涨跌幅 |
| 因子明细 | 各因子得分 |

#### 4.3.3 导出

- **CSV**：单次结果导出为 CSV 文件
- **Excel**：多策略汇总导出为多 Sheet Excel

导出文件保存在服务端 `data/csv/` 或 `data/excel/`，浏览器自动下载。

#### 4.3.4 进入回测

点 **回测** 按钮，跳转到回测视图，对该策略进行历史数据回测。

---

### 4.4 信号中心 Tab 🔔

查看预警信号，查看详情，重推信号。

#### 4.4.1 信号列表

| 字段 | 说明 |
|------|------|
| 时间 | 信号触发时间 |
| 类型 | 信号类型（如涨停预警 / 趋势突破） |
| 策略 | 触发策略 |
| 股票 | 股票代码与名称 |
| 状态 | 推送状态（已推送 / 部分成功 / 失败） |
| 通道 | 推送通道徽章（通达信/Web/飞书/CSV） |
| 严重度 | 严重 / 警告 / 信息 |

#### 4.4.2 筛选

- 按 **策略** 筛选
- 按 **通道** 筛选
- 按 **状态** 筛选
- 按 **时间范围** 筛选

#### 4.4.3 信号详情抽屉（R7 新增）

**点击任意信号行** 打开右侧抽屉：

| 区域 | 内容 |
|------|------|
| 顶部 | 信号类型 + 时间 + 策略 |
| 基本信息 | 信号 ID / 股票代码 / 股票名称 / 策略 ID / 推送状态 / 严重度 |
| 推送通道 | 各通道徽章 + 推送结果 |
| 信号内容 | 完整预警文本 |
| Snapshot | 信号触发时的行情快照（JSON 树形展示，带语法高亮） |
| 底部操作 | **重新推送** / **复制 JSON** / **关闭** |

**Snapshot JSON 树**：
- key 琥珀色 / string 绿色 / number 蓝色 / boolean 紫色 / null 灰色
- 支持滚动查看

#### 4.4.4 重新推送

1. 打开信号详情抽屉
2. 点底部 **重新推送**
3. 系统按原通道重新发送预警
4. 通知中心显示结果

### 4.5 板块管理 Tab 🗂️

管理策略对应的通达信板块，维护成份股。

#### 4.5.1 板块列表

每个策略对应一个板块（如 `ZD_DBQZT01` 打板求涨停选股）：

| 字段 | 说明 |
|------|------|
| 板块代码 | 如 `ZD_DBQZT01` |
| 板块名称 | 如「打板求涨停选股」 |
| 关联策略 | 如 `dbqzt` |
| 成份股数 | 当前板块内股票数量 |
| 更新时间 | 最后回写时间 |

#### 4.5.2 查看成份股

点板块的 **查看** 按钮，展开成份股列表（代码 / 名称 / 现价 / 涨跌幅）。

#### 4.5.3 刷新板块

点 **刷新** 按钮，重新从策略选股结果回写到通达信板块。

> ⚠️ Real 模式下需通达信终端已启动。

#### 4.5.4 批量导出

点 **导出全部**，将所有板块的成份股导出为 Excel（多 Sheet）。

### 4.9 推送通道配置 ⚙️

|------|------|----------|
| **通达信预警** | 推送到通达信客户端（半自动交易） | 开启 |

   - 签名密钥

5. 点 **测试** 发送测试消息
6. 点 **保存**

#### 4.9.3 测试通道

每个通道卡片有 **测试** 按钮，点击发送测试消息，验证配置是否正确。

## 五、常见操作指南

### 5.1 如何运行一个策略选股？

```
策略管理 Tab → 找到策略 → 确认已启用 → 点 ▶️ 运行
→ 等待完成 → 切到选股结果 Tab 查看
```

### 5.2 如何导出选股结果？

```
选股结果 Tab → 找到记录 → 点 导出 → 选 CSV 或 Excel
→ 浏览器自动下载
```

### 5.3 如何创建一个新策略？

```
策略管理 Tab → 找到相似策略 → 点 📋 复制
→ 填新 ID / 名称 / emoji → 确认 YAML 预览 → 确认创建
→ 新策略默认禁用 → 开启启用开关 → 运行验证
```

### 5.4 如何调整策略阈值？

> ⚠️ 当前版本策略阈值修改需编辑 YAML 文件（由管理员操作）。

```
管理员编辑 strategies/strategy_xxx.yaml
→ 修改 factors[].params 或 scoring.expression
→ 点顶部 🔄 热加载配置（或等 2 秒自动生效）
→ 运行策略验证效果
```

### 5.5 如何查看某个信号的详情？

```
信号中心 Tab → 点击信号行 → 右侧抽屉打开
→ 查看 Snapshot JSON 树 → 点 复制 JSON 可复制到剪贴板
→ 如需重发，点 重新推送
```

### 5.6 如何回测一个策略？


## 六、常见问题（FAQ）

### Q1：页面打开是白屏？

**可能原因**：服务未启动 / 网络问题
**解决**：
1. 刷新页面
2. 联系管理员检查服务状态
3. 通过预览面板访问（勿用 localhost 地址）

### Q2：运行策略后没有结果？

**可能原因**：策略未启用 / 股票池筛选过严 / 数据源问题
**解决**：
1. 确认策略启用开关为绿色
2. 检查策略的 `universe` 和 `pool.expression` 筛选条件
3. Mock 模式下数据有限，可能选不出股
4. 切换 Real 模式（需通达信终端）

### Q3：信号没有推送？

**可能原因**：通道未开启 / 策略未配置通道 / 配置未热加载
**解决**：
1. 顶部 ⚙️ 检查通道开关
2. 确认策略 YAML `monitor.alert_conditions[].channels` 引用了正确通道
3. 点 🔄 热加载配置
4. 信号中心找到该信号 → 详情 → 重新推送

### Q4：导出的 CSV/Excel 在哪？

**答**：浏览器自动下载到下载目录。服务端副本在 `data/csv/` 和 `data/excel/`。

### Q5：回测结果是真实的吗？

**答**：当前 Mock 模式下回测用确定性 mock 价格（基于股票代码 hash 生成），**仅用于功能演示**。Real 模式下会接入真实历史数据（规划中）。

### Q6：资金流向数据是真实的吗？

**答**：Mock 模式下从 V8 样本 CSV 提取真实字段（如隆扬电子主力净流入 19836 万）。Real 模式下从 `tq.get_more_info` 获取实时资金流。

### Q7：通知刷新就没了？

**答**：已知限制，通知仅存浏览器内存。规划中：localStorage 持久化。

### Q8：如何删除一个策略？

```
策略管理 Tab → 关闭策略启用开关 → 点 🗑️ 删除
→ 弹窗输入完整策略 ID 确认 → 确认删除
```

> ⚠️ 启用中的策略不可删除，必须先禁用。

### Q9：策略阈值改了不生效？

**解决**：
1. 点顶部 🔄 热加载配置
2. 或等 2 秒自动重载
3. 仍不生效检查 YAML 语法（缩进 / 引号）
4. 联系管理员看 FastAPI 日志

### Q10：全局搜索快捷键冲突？

**答**：可点击顶部搜索按钮代替快捷键。

### Q11：如何反馈 Bug 或建议？

见下一章「问题反馈渠道」。

### Q12：K 线图数据是真的吗？

**答**：当前 Mock 模式生成模拟 K 线。Real 模式将接入 `tq.get_market_data`（规划中）。

### Q13：多个策略能同时运行吗？

**答**：可以。点顶部 **▶️ 运行全部** 一键运行所有已启用策略，或逐个点运行。

### Q14：信号详情的 Snapshot JSON 看不懂？

**答**：Snapshot 是信号触发时的行情快照。关键字段：
- `run_id`：触发的选股运行 ID
- `strategy_id`：策略 ID
- `top_picks`：选股结果前 N 名
- `duration_sec`：执行耗时

颜色含义：key 琥珀 / 字符串绿 / 数字蓝 / 布尔紫 / null 灰。

### Q15：如何切换 Mock / Real 模式？

**答**：由管理员修改 `config/app.yaml` 的 `app.adapter_mode`（mock / real），需重启 FastAPI。终端用户不可操作。

## 八、术语表

| 术语 | 解释 |
|------|------|
| **策略** | 一套选股规则（YAML 定义），包含因子组合 + 评分公式 + 预警条件 |
| **因子** | 选股维度（如 5 日动量 / 换手率 / 量比），26 个内置因子 |
| **评分** | 策略对股票的综合打分，越高越优先 |
| **universe** | 股票池筛选（排除 ST / 停牌 / 新股） |
| **pool** | 策略专属股票池筛选表达式 |
| **清洗** | 数据预处理（去极值 / 缺失值处理） |
| **快照** | 某时刻的行情数据（价格 / 量 / 资金流） |
| **Snapshot** | 信号触发时的完整行情快照（JSON） |
| **板块** | 通达信自定义板块，策略选股结果可回写 |
| **Mock 模式** | 沙箱开发模式，用 V8 CSV 样本数据 |
| **Real 模式** | 生产模式，调用通达信 tqcenter API |
| **适配器** | 数据源抽象层（MockAdapter / RealAdapter） |
| **热加载** | 修改 YAML 配置后无需重启即生效 |
| **通道** | 推送通道（通达信 / Web / 飞书 / CSV） |
| **重推** | 重新推送历史信号 |
| **回测** | 用历史数据验证策略表现 |
| **夏普比率** | 风险调整后收益，越高越好 |
| **最大回撤** | 历史最大亏损幅度，越小越好 |
| **胜率** | 盈利交易占比 |
| **Alpha** | 超额收益（相对基准） |
| **Beta** | 相对市场的波动倍数 |
| **主力净流入** | 大单买入净额（万元） |
| **大买占比** | 大买单占总成交比例 |
| **换手率** | 当日成交量 / 流通股本 |
| **涨停** | A 股当日涨幅达 10%（ST 5%） |
| **打板** | 追涨停交易策略 |
| **主升浪** | 趋势加速上涨阶段 |
| **错杀** | 非理性下跌后的反弹机会 |
| **弱转强** | 前弱后强的反转信号 |
| **XTransformPort** | Caddy 网关端口转发参数（内部使用） |
